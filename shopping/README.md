# おかいものメモ 🐱🧺

主婦のための、**かわいい**お買い物リスト PWA。
スマホ単体で完結（端末内保存・ログイン不要・無料・オフラインOK）。
よく買うものはワンタップ、店内は売り場順でスイスイ。

> 純フロントエンド（HTML / CSS / バニラ JS）。ビルド不要・サーバー不要。

---

## ✨ できること（Phase 1 / MVP）

- **買うものリスト** … 追加 / 数量変更 / チェックで「カゴに入れた」/ 削除。完了品は下に薄く移動。全品チェックすると **「🎉コンプリート！」** で激励し、「🛍️次のお買い物に行く？」ボタンで完了品を片付けて次の買い物にすぐ移れます。
- **予測入力（学習）** … 一度書いたものを覚えて、入力途中でサジェスト。よく買うものは「⭐よく買う」タブからワンタップ追加。単位・単価も前回値を自動補完。
- **売り場順の並び替え** … 商品名から自動でカテゴリ分類し、野菜→お肉→お魚→乳製品…の売り場順にカード表示。
- **予算・合計金額の概算** … 品ごとの単価を記憶してカゴ合計をリアルタイム表示。予算バー＋オーバー時のやさしい警告。商品ごとに **「前回¥◯◯」** を表示し、値上がりは↑・値下がりは↓で一目で分かります。
- **かわいいUI ＋ PWA** … パステル配色・丸ゴシック・子ねこマスコット・チェック時の「えらい！」ご褒美演出。ホーム画面に追加＆オフライン動作。

データはすべて端末内（`localStorage`）に保存され、外部に送信されません。

## ✨ できること（Phase 2 / 便利機能）

> 🏠 **起動時のホームは「🍳献立」タブ**。作りたい料理から材料をすぐ検索できます。

- **作る献立 → 材料を自動で買い物リストへ（「これを作る」）** … 「🍳献立」タブで料理名を入力して**「決定」**を押すと、必要な材料を自動で検索・表示。**「これを作る」**を押すと、**お買い物リストの上部に「作る献立」**が並び、**その下に必要な材料**が入ります。**登録にない料理も端末内エンジンで材料を自動推定**するので、同じく「これを作る」でOK。あいまい検索対応（近い名前・カタカナ/ひらがなのゆれ・タイプミス・材料名）。**似ているメニュー**もその場で選べます。
- **作る人数で量を簡易調整・複数献立の材料はまとめる** … 「作る献立」ごとに**人数（人前）**を入れると材料の量を簡易スケール。複数の献立を入れると**被る材料は1つにまとめて**表示。不要になった献立は**✕で削除**でき、材料も自動で組み直します。
- **ズボラメシ（時短・節約）** … 献立の隣の「😋ズボラメシ」タブに、**米類・麺類・一人暮らし飯・1000円以下節約飯**のかんたんメニューを多数用意。同じく「これを作る」で材料がリストに入ります。
- **「作った」は写真で自動カウント＋称号** … 料理写真を献立カードにのせると、**自動で「作った」が+1**（手動ボタンは廃止）。間違えたらカードの**「↩︎戻す」**で1つ戻せます。**作った献立の種類が3つ増えるごとにスタンプ**がたまり**称号**がランクアップ（全10段階）。称号が上がるとヘッダー上部のバッジが**煌びやかに変化**。**次の称号名は獲得するまで「？？？」で伏せ**、手に入れた瞬間に公開されます。
- **料理写真のSNSシェア** … のせた写真を **X・Instagram・Facebook** にシェア。スマホは Web共有（`navigator.share`）で各アプリへ直接投稿、PCは画像保存＋X/Facebook導線にフォールバック（写真は端末内に圧縮保存）。
- **自分の献立** … 名前＋材料で保存すると献立の上部に表示され、「これを作る」で材料がリストに入ります。
- **買い忘れリマインド / 在庫** … カゴに入れた品を自動で記録し、**買う周期を学習**。周期を過ぎると「🔔 そろそろ買い忘れ？」をリスト上部に提案（タップで再追加）。周期は手動でも調整でき、設定タブの「買い忘れ・在庫」で一覧管理できます。

## ✨ できること（Phase 3 / お店検索・効率ルート）※オンライン時のみ

- **近くのお店検索** … 「🗺️お店」タブで現在地（ブラウザの位置情報・許可制）から、近くのスーパー・ドラッグストア等を **OpenStreetMap（Overpass API・無料/キー不要）** で検索し、近い順に表示。地図は **Leaflet＋OSMタイル** で表示します。**位置情報が許可されていないときは、端末（iPhone / Android / PC）ごとの設定手順と再試行ボタン**を案内します。
- **効率よく回るルート** … 行きたいお店を保存→チェックして「効率ルートを計算」すると、現在地から **最近傍法** で回る順番と移動距離の目安を提案し、地図に線を引きます。さらに **店内の回り方（売り場順）** も、いまのお買い物リストから売り場の動線順に提案します。

> お店検索・地図はオンライン時の拡張で、オフラインのコア機能（リスト・予測・売り場順・予算・献立・在庫）とは分離しています。位置情報は端末内で使うだけで外部送信しません（検索クエリのみ OpenStreetMap に送られます）。

```
shopping/

├── app.js           … リストCRUD・localStorage・予測学習・予算・在庫・演出

├── recipes.js       … 献立（レシピ）→ 材料 の初期データ
├── stores.js        … お店検索（Geolocation/Overpass）・地図・効率ルート
├── manifest.json    … PWAマニフェスト
├── sw.js            … Service Worker（オフライン・キャッシュ）
├── favicon.svg / favicon-16.png / favicon-32.png … ファビコン（子ねこ）
├── icons/
│   ├── icon-192.png

└── README.md
```

### データモデル（localStorage キー）
| キー | 内容 |
|------|------|
| `okaimono.items` | 今のリスト `[{id,name,category,qty,unit,price,prevPrice,checked,fromPlan,addedAt}]`（`fromPlan` は献立から自動生成された材料） |
| `okaimono.plans` | これから作る献立 `[{id,name,emoji,servings,items}]`（材料は人数でスケールし重複はまとめる） |
| `okaimono.catalog` | 学習辞書 `[{name,category,count,lastUsed,defaultUnit,defaultPrice}]` |
| `okaimono.staples` | 「いつもの」定番品（ワンタップ追加用） |
| `okaimono.inventory` | 在庫/購入周期 `[{name,lastBought,intervalDays,history,manual}]` |
| `okaimono.recipes` | ユーザー自作の献立 |
| `okaimono.cooks` | 献立ごとの「作った回数」 `{ recipeId: {count,last} }`（称号の判定に使用） |
| `okaimono.photos` | 献立ごとの料理写真 `{ recipeId: dataURL }`（長辺1080px・JPEG圧縮） |
| `okaimono.stores` | 保存したお店 `[{id,name,lat,lon,type}]` |
| `okaimono.budget` | 予算設定 |
| `okaimono.settings` | 売り場順・合計表示・買い忘れ提案のON/OFF・獲得済み称号レベル など |

## 🏃 ローカルで動かす

ブラウザのセキュリティ上、Service Worker は `file://` では動かないため、簡易サーバで開きます。

```bash
cd shopping
python3 -m http.server 8000
# → ブラウザで http://localhost:8000/ を開く
```

スマホで確認する場合は、PCと同じWi-Fiで `http://<PCのIP>:8000/` を開きます。

---

## 🚀 デプロイ（無料・ビルド不要）

静的ファイルだけなので、そのまま公開できます。

- **Cloudflare Pages** … リポジトリを連携。Build command なし、出力ディレクトリに `shopping`（またはルート）を指定。
- **GitHub Pages** … リポジトリの Settings → Pages で公開ブランチを指定。`/shopping/` 配下のURLで開く。

> PWA（ホーム画面に追加・オフライン）には **HTTPS** が必要です。上記サービスは標準でHTTPS対応です。

---

## 📱 スマホで“アプリのように”使う（ホーム画面に追加）

公開URL（HTTPS）をスマホのブラウザで開くと、アプリ内で案内バナーが出ます。

- **iPhone（Safari）** … 共有ボタン <kbd>⬆️</kbd> →「ホーム画面に追加」。全画面で起動し、ステータスバー対応・ノッチ/ホームバーの余白も調整済み。
- **Android（Chrome）** … バナーの「追加」ボタン、またはメニュー →「アプリをインストール」。

ホーム画面アイコンは子ねこ🐱。一度追加すれば、オフラインでもコア機能（リスト・予測・売り場順・予算）が動きます。

> スマホ最適化：タップ領域は指で押しやすいサイズ、入力欄は16px以上で iOS の自動ズームを防止、プルリフレッシュのバウンスも抑制しています。

## ✅ 動作確認（Verification）

- **永続化**：品を追加 → リロード → 残っていること。再入力でサジェスト候補に出ること。
- **売り場順**：野菜・肉・日用品を混ぜて追加 → カテゴリ順に並ぶこと（設定でON/OFF切替）。
- **予算**：単価入力 → 合計とバーがリアルタイム更新、予算超過でやさしい警告。
- **PWA**：`manifest.json` / `sw.js` 認識・「ホーム画面に追加」可・機内モードでコア機能が動くこと。
- **献立**：献立をタップ → 材料がまとめて追加。自作の献立を保存 → 呼び出せること。
- **買い忘れ/在庫**：カゴに入れる → 記録され、周期を過ぎると「そろそろ」提案が出ること。
- **お店検索/ルート**：位置情報を許可 → 近隣店がヒット → 保存 → 「効率ルートを計算」で近い順の回り方と距離が出ること（オンライン・実機推奨）。
